What is Webswing?

Webswing is a web server, that allows you to run any swing application inside your web browser using only pure HTML5.

What are the benefits of using Webswing?

  • Webswing offers the most secure way of providing rich-client experience through web.
  • Webswing works with all major browsers (incl. IE9, Chrome and Firefox).
  • Webswing provide 100% same user experience for your swing application as on desktop.
  • You don't need to change single line of source code of your swing application.
  • With Webswing your clients don't need JRE installed, so you don't need to worry about security issues in JVM.
  • With Webswing you don't need to think about distributing the swing application to clients, only deploy it on server.

How Webswing works?

Webswing acts as a regular web server, which listens on a specified port. When the webswing page is loaded in web browser, javascript opens a web socket connection and sends requests to start the swing application. Server then starts a new JVM where the swing application is loaded by a special classloader and started. All the magic is then done in runtime by the special classloader, which intercepts the painting routine of the swing application. When swing application repaints its content, the classloader takes care of rerendering the content in web browser, which means we are able to provide 100% same behavior and look of the application in the web browser (incl. animations and look&feel change). On the browser side we are using HTML5 canvas for rendering the swing application. For controlling the application mouse and keyboard events are sent to server. In summary, we can say webswing works (and performs) like a remote dektop for swing applications.

How to use Webswing?

Easiest way to try Webswing on your computer is to run demo application. For that you only need to download the latest distribution, unzip it, execute start.bat or start.sh open browser and load http://localhost:8080

If you want to try your own swing application, you need to adjust the starting command in the script. Webswing offers following command line options for this: 

 -h,--host         Local interface address where the web server will listen.(localhost)
 -p,--port         Port where the web server will listen.(8080)
 -c,--clients      Maximum number of simultaneous swing connections.(10)
 -m,--mainClass    Swing application main class.
 -a,--args         Commandline arguments for swing application.
 -v,--vmargs       JVM arguments for swing application.
Main class name is the only required option to successfully start Webswing. But you need to place your application and its required libs to swinglib directory in flat structure (no sub-directories).
